onMessage

The onMessage method is called when a message is handled by the server.

onMessage(
  _fromClientId: string,
  _toClientId: string,
  _remainder: string,
): void
TypeDescription
_fromClientIdstring
_toClientIdstring
_remainderstring
returnsvoid

This has no return value.

Note that this method is potentially called very frequently, and should be performant. A pathId is not passed as an argument, since it has a small cost to provide by default. You can use the getPathId method yourself if that information is needed.

Since

v5.4.0
